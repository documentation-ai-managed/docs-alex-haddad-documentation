## Release History

<Update label="2024-10-15" description="v1.2.0" tags={["feature", "improvement"]}>

## New Features

- Added secure file uploads up to 500MB with client-side encryption.
- Introduced team-based access controls for shared secrets.

## Improvements

- Reduced share expiration setup time by 40%.
- Enhanced UI for password generator with strength indicators.

## Bug Fixes

- Fixed token refresh issues during long sessions.
- Resolved cross-browser compatibility for drag-and-drop files.

</Update>

## New Features

- Integration with SSO providers like Okta and Auth0.

## Bug Fixes

- Patched potential XSS in share preview links.
- Corrected decryption errors on mobile devices.

## Improvements

- Optimized database queries for faster secret retrieval.

</Update>

## New Features

## Upgrading to Latest Version

Follow these steps to upgrade Crivvi safely and apply any breaking changes.

<Steps>
  <Step title="Backup Data" icon="database">
    Export all shares from your dashboard.
  </Step>
  <Step title="Update SDK" icon="download">
    Install the latest SDK version.
    
    <CodeGroup tabs="npm,yarn">
    ````bash
    npm install @crivvi/sdk@latest
    ````
    ````bash
    yarn add @crivvi/sdk@latest
    ````
    </CodeGroup>
  </Step>
  <Step title="Migrate Auth" icon="settings">
    Update your integration code for new token format.
    
    <Tabs>
      <Tab title="JavaScript" icon="code">
        ```javascript
        const client = new CrivviClient({ token: '{CRIVVI_TOKEN}' });
        const share = await client.createShare({ secret: 'my-password' });
        ```
      </Tab>
      <Tab title="Python" icon="python">
        ```python
        from crivvi import Client
        client = Client(token='{CRIVVI_TOKEN}')
        share = client.create_share(secret='my-password')
        ```
      </Tab>
    </Tabs>
  </Step>
  <Step title="Test Shares" icon="check-circle">
    Create test shares and verify decryption.
  </Step>
</Steps>

<Callout kind="tip">
  For breaking changes like auth updates, review the migration guide in your account settings.
</Callout>

<Expandable title="Detailed Migration Notes" default-open="false">

If you encounter issues with `{CRIVVI_TOKEN}`, regenerate it from the API keys section. Legacy query params are deprecated and will stop working after 2024-11-01.

</Expandable>
